  - `Light00Side/horuspublicrelay` (Cloudflare Worker relay)

## Data flow / scaling

- DO cache model adopted as source for public clients.
- PM2 bridge on VPS now pushes snapshots into DO every 1000ms.
- Snapshot caps set to reduce bandwidth:
  - signals = 50
  - incidents = 30
- `horuspublic` switched from polling-heavy component fetches to single WS snapshot stream (`/ws`) consumption for consistency and lower request load.

## J7 / signals

- Public live feed path had instability with direct in-worker J7 ingestion.
- Durable Object WS side-route added on relay.
- Current intended source of truth for public clients is DO snapshot broadcast (not per-component direct upstream fetching).
